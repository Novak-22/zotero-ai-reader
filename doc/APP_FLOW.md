# Application Flow

## 用户交互流程

### 1. 插件启动
```
用户打开Zotero → 加载插件 → 注册Reader Pane Sections
```

### 2. PDF阅读流程
```
用户打开PDF → 提取文本 → 检测段落边界
    ↓
生成AI目录 → 显示在左侧边栏
    ↓
逐段生成总结 → 显示在页面旁
```

### 3. 对话交互流程
```
用户选中文字 → 点击对话图标
    ↓
打开右侧对话栏 → 输入问题
    ↓
发送至LLM → 显示回复
    ↓
累积上下文 → 支持多轮对话
```

### 4. 对话历史
```
关闭PDF → 保存对话到SQLite
再次打开 → 从数据库恢复对话历史
```

## 数据流

```
PDF文件 → Zotero API提取文本 → 段落分割
    ↓
段落文本 → LLM API → 中文总结
    ↓
总结 + 原文 → 存储 + 显示

用户输入 → LLM API → 回复
    ↓
对话历史 → SQLite存储
```

## 状态管理

| 状态 | 描述 |
|------|------|
| IDLE | 等待用户打开PDF |
| LOADING | 提取PDF文本中 |
| TOC_GENERATING | 生成目录中 |
| SUMMARIZING | 生成段落总结 |
| CHATTING | 对话中 |
| ERROR | 发生错误 |
